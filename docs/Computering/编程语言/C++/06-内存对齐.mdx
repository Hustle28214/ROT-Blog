内存对齐（Memory Alignment）是计算机系统中的一个重要概念，尤其在底层编程（如 C/C++）中，它直接影响数据结构的内存布局、程序性能和可移植性。

内存对齐是指将数据存储在内存地址上时，要求该地址是某个值（通常是数据类型的大小或其倍数）的整数倍。例如，一个 4 字节的 int 类型变量如果存储在地址 0x1000（4 的倍数），则称它是“对齐”的；如果存储在 0x1001，则是“未对齐”的。

在结构体中，编译器会在成员之间插入未使用的字节（称为填充，padding），以保证每个成员都满足其对齐要求，同时使结构体整体大小符合对齐规则。

:::info
内存对齐主要是出于硬件和性能的考虑。许多处理器（如 x86、ARM）访问对齐的数据时效率最高，而访问未对齐的数据可能需要多次内存访问、异常处理或根本不支持。例如，某些 RISC 处理器在访问未对齐的地址时会触发硬件异常，操作系统不得不通过软件模拟来处理，极大降低性能。

对齐的数据可以保证原子读/写，对于多线程环境下的锁自由编程至关重要。对齐的数据可以更好地利用缓存行，减少缓存未命中和总线通信。

不同硬件平台的对齐要求不同，遵循对齐规则可以使代码在不同架构上正确运行。
:::

## 基本概念

对齐值（Alignment）：一个类型要求其起始地址必须是的倍数。例如，`int` 通常要求 4 字节对齐，所以它的对齐值是 4。

自然对齐：将类型的大小作为其对齐值，例如 `int` 大小为 4，对齐值为 4。但有时编译器可能会根据平台调整对齐值。

有效对齐值：编译器实际使用的对齐值，可能受编译选项或 `#pragma pack` 影响。

| 类型            | 大小（字节） | 对齐值（字节）               |
| --------------- | ------------ | ---------------------------- |
| `char`          | 1            | 1                            |
| `short`         | 2            | 2                            |
| `int`           | 4            | 4                            |
| `float`         | 4            | 4                            |
| `double`        | 8            | 8（32位可能为4）             |
| `long long`     | 8            | 8                            |
| 指针（`void*`） | 4/8          | 4/8                          |

注意：在 32 位系统上，double 可能只要求 4 字节对齐（例如 Windows 上的 Visual C++），但在 Linux 上通常要求 8 字节对齐。这些细节依赖于平台和编译器。
