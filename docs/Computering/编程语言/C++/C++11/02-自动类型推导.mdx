## 1. auto关键字

auto用于变量的类型推导，根据初始化表达式的类型来推导变量的类型，常用于简化代码和处理复杂类型。

```cpp
auto a = "hello"; // 自动推导出字符串
auto b = 2; // 自动推导出int
```

在写算子优化求解等问题时，`auto`发挥着非常关键的作用，特别是构建张量，它能在`<static_cast>`满天飞的代码里自动省去很多麻烦。

`auto`的原理类似于模板参数推导。



### 1.1 auto的限制

1. 必须初始化变量
2. 不能用作函数参数
3. 不能用于数组
4. (但可以用于指针)
5. 无法推导出模板参数
6. 不能用于非静态成员变量的直接类型推导（C++17 前）
7. 当用 auto 定义数组变量时，数组名会退化为指针

```cpp
int arr[5];
auto a = arr;         // a 的类型是 int*（退化为指针）
auto& b = arr;        // b 的类型是 int(&)[5]（保留数组信息）

auto s = "hello";     // s 的类型是 const char*，而非 const char[6]
```

8. 不能用于位域（bit-field）

```cpp
struct S {
    auto b : 1;       // 错误：位域不能使用 auto
};
```

9. 当使用花括号 `{}` 初始化时，auto 会推导为 std::initializer_list，而不是数组或其他容器。
```cpp
auto x = {1, 2, 3};   // x 的类型是 std::initializer_list<int>
auto y{1};            // C++17 后：y 为 int；C++11/14 中也是 std::initializer_list<int>
auto z = {1};         // 始终是 std::initializer_list<int>
```


10. 默认情况下，auto 会忽略顶层 const 和引用，需要显式添加才能保留。

```cpp
const int ci = 10;
auto a = ci;            // a 是 int（顶层 const 被忽略）
auto& b = ci;           // b 是 const int&（底层 const 保留）
auto c = &ci;           // c 是 const int*（指针指向 const）
const auto d = ci;      // d 是 const int5 

// 若需保留表达式的完整类型（包括引用），应使用 decltype(auto)。

int x = 0;
int& ref = x;
const int& cref = x;

auto v1 = ref;                // v1: int
decltype(auto) v2 = ref;      // v2: int& (ref 是左值引用，所以 v2 也是 int&)

auto v3 = cref;               // v3: int (顶层 const 和引用被剥离)
decltype(auto) v4 = cref;     // v4: const int&

// 函数返回类型中使用
int& getRef(int& a) { return a; }
auto ret1 = getRef(x);        // ret1: int (引用被剥离)
decltype(auto) ret2 = getRef(x); // ret2: int& (保留引用)
```

11. 不能用于函数参数中的数组大小。在函数参数中，不能使用 auto 指定数组大小，如 `void f(int arr[auto])` 非法（但可通过模板实现）。

:::info
C++14 允许函数使用 auto 作为返回类型（后置或前置），但要求：

1. 所有 return 语句推导出的类型必须一致。

2. 不能用于虚函数（虚函数需要固定签名）。

3. 递归函数中，在递归调用前至少有一个 return 语句以确定类型。

4. 返回类型为 auto 的函数在使用前必须可见（不能仅靠前向声明）。

5. 不能返回数组或函数类型（但可返回指针或引用）。
```cpp
auto func() { return 42; }        // 正确，返回 int
auto fib(int n) {                  // 递归需谨慎
    if (n <= 1) return n;          // 必须有此句才能推导
    return fib(n-1) + fib(n-2);    // 推导为 int
}
```
:::



## 2. decltype关键字

```cpp
decltype(entity)       // entity 是一个标识符（变量、函数等）
decltype(expression)   // expression 是任意表达式

int a = 10;
const int& b = a;
decltype(a) x;          // int
decltype(b) y = a;      // const int&（必须初始化）

// 其他类型
int a = 0;
int* p = &a;

decltype(a)     d1;    // int
decltype((a))   d2 = a; // int&，必须初始化
decltype(*p)    d3 = a; // int&（解引用指针产生左值）
decltype(++a)   d4 = a; // int&（前置自增返回左值）
decltype(a++)   d5;     // int（后置自增返回纯右值）
decltype(42)    d6;     // int（字面量是纯右值）
decltype(std::move(a)) d7 = 0; // int&&（std::move(a) 是将亡值）
```

:::info
`decltype`在C++14中可以实现`decltype(auto)`, 用于让变量或返回类型的推导采用 decltype 的规则，而不是 auto 的规则。通常用于转发函数或完美返回。

```cpp
decltype(auto) f1() { return x; }      // 返回类型为 x 的声明类型
decltype(auto) f2() { return (x); }    // 返回类型为 int&（注意括号！）
```
:::

然而如果我们要用auto推导函数的返回类型，会这样写：

```cpp
template<typename X, typename Y, typename Z>
X AddTwoItems(Y y,Z z){
    return y+z;
}
// 以下注释掉的写法是不合法的：
//decltype(y+z) AddTwoItems(Y y,Z z);

```
