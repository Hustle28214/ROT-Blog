## 柔顺控制简介

我们先来介绍一下柔顺是什么：

机器人在与环境或人发生接触时，能够主动或被动地顺应外部作用力，而不是刚性对抗。

再来介绍一下被动柔顺控制和主动柔顺控制：

假设你现在在ROS的执行脚本中写死了行进轨迹，机械臂按轨迹决定位姿。但现在我往轨迹中间放了一个障碍物，因为路径是写死的，所以机械臂肯定会碰到这个障碍物。

但是碰到之后该怎么办呢？大力出奇迹扬了障碍物？那投产肯定是不行的，所以我们要想些办法让碰撞的影响最小化。

有人出主意，说给可能碰撞的部位，比如执行器加个弹簧，再加个阻尼器，这样我碰到障碍物的时候，弹簧会吸收能量压缩，阻尼器会耗散能量阻尼，从而减少碰撞的影响。

这是被动柔顺控制。它并不是个很好的策略，因为非常影响控制的精度，导致控制结果与期望结果的差异增大。

而且这是不可调节的，因为弹簧的刚度和阻尼器的阻尼系数是固定的，不能根据实际情况进行调整。

有人出主意，说你把机械臂末端的刚度弄成可以调节的，最好再弄个位控的增益，这样我就能更好的应对不同的碰撞情况。

这还是被动柔顺控制，但是这次更好一些，因为我们多了一些能够控制的东西，使得影响更小了。

有人说，那照你这么说，我搞个大一统不就好了，怎么做呢，我把整个系统抽象出来，一个“质量-弹簧-阻尼”系统。

这就是主动柔顺控制。其实与其说这是个机器人界的问题，不如说这是个力学和控制界的问题。



## 主动柔顺控制

主动柔顺控制大致可分为三种：

导纳控制、阻抗控制、力位混合控制。

力位混合控制是一种将力控制（导纳控制）和位置控制（阻抗控制）相结合的控制策略。常用于机械臂、机器人的控制上面。不过因为这个应用方向比较新，也是去年才拿的CoRL最佳论文奖，所以目前还有很大的发展空间，可以和传统控制方法再权衡。

在介绍这个概念之前，我们要先介绍一下导纳控制和阻抗控制。顺便复习一下材料力学。

为什么要介绍导纳控制和阻抗控制？因为如果你对输出力进行控制，那么你就无法控制位置偏差；反之，你无法控制位置。

带着这个结论，我们认识一下原因。



### 力控制(导纳控制)

我们先来复习一个初中就学过的物理概念：

$$
F = kx
$$

这就是胡克定律。这个公式表示在**弹性接触**中，力与位置的关系。$k$ 是一个常数，$x$ 是位置。

$k$实际上也叫刚度系数，它表示了弹性接触的刚度。越大越刚，越小则越有弹性。

刚度是可以测算的，同样也是通过这个公式的变式来测算，也就是**物体所受应力与产生的形变量的比值**。

让我们再来看一个场景，事实上，我们可以在工程中对物体做这样理想的场景假设，那就是把物体间的接触都抽象成**阻尼质量弹簧**模型。

这个模型也叫做单自由度振动系统。

:::info
“阻尼质量弹簧”模型 假设：

- 质量集中（质点或刚体）

- 线性弹性力（$F=kx$）

- 线性阻尼力（$F=c\dot{x}$）

- 小变形、无永久变形、无记忆效应

实际上要做那种工程热物理计算的大工程非常复杂，弹塑性、纯塑性、粘弹性、断裂、粘着、摩擦滑动等的场景都可能出现。但是做机器人就还行这个假设可以用。
:::

一个标准**质量-弹簧-阻尼系统**由三个元件并联（或串联）构成：

- **质量 $m$**：储存动能，使系统有惯性（对应牛顿第二定律 $F = m a$）。
- **弹簧**：产生**弹性恢复力** $F_s = k \cdot x(t)$，其中 $k$ 是**刚度**。弹簧储存势能。
- **阻尼器**：产生**阻尼力** $F_f = c \cdot \dot{x}(t)$，其中 $c$ 是阻尼系数（单位 N·s/m），耗散能量。


这个模型是怎么抽象得出的呢？

任何实际物体在受力时都会表现出三种基本响应：

1. 惯性：物体抵抗速度变化的倾向（因质量存在）。

> 例：推一辆小车，它不会瞬间达到最终速度，而是逐渐加速。

2. 弹性：物体受力变形，力卸除后恢复原状的趋势（因分子间作用力）。
> 例：压一个弹簧或一块橡胶，它会反弹。

3. 耗散：物体运动时能量转化为热，运动逐渐衰减的趋势（因内部摩擦、粘性等）。
> 例：在油中搅拌叶片，速度越快阻力越大；振动在空气或材料中会慢慢停止。

集总参数（Lumped Parameter）是一种理想化的物理与工程建模方法。它将实际物理系统中分布在空间各处的参数（如电阻、电容、电感、质量、热容等）集中在有限的、离散的几何点上进行处理，

这里就用了集总参数方法。假设这三种特性可以分别集中到三个独立的理想元件上，而不是连续分布在整个物体中。

这三个元件用刚性的无质量连接组合在一起，代表真实物体的力学行为。

再然后，决定它们的连接方式，就是并联或串联，或者混联。最常见的做法是，弹簧和阻尼器并联，质量串联进来。

质量块一端自由（受外力 $u(t)$），另一端连接一个弹簧和一个阻尼器，弹簧和阻尼器的另一端固定。

此时质量块的位移 $x(t)$ 同时拉伸弹簧（产生 $kx$）和拉动阻尼器（产生 $c\dot{x}$）。

这种并联结构能同时产生与位移成正比的恢复力和与速度成正比的阻尼力。

我们可以用力学语言描述这个系统。

$$
M\ddot{x}+F_s+F_f=u(t)
$$

$$
M\ddot{x}+C\dot{x}+Kx=u(t)
$$


则：

$$
\ddot{x} = M^{-1}(u(t)-C\dot{x}-Kx)
$$

对$\ddot{x}$两次积分，得到位置。

则引入到控制器中，可以通过控制位置，来实现控制力。

这也叫做**导纳控制**。

导纳模型：

$$
F = M\ddot{x}+C\dot{x}+Kx
$$

我们需要测量的东西是力和位置，需要控制的东西是位置。当期望力和实际力产生偏差，调整位置（实际需要转换为关节空间的位置）来控制，这里需要用到位置环。位置环也是用的相当多的一环，导纳控制的成本因此稍低。

调节$K$可以改变弹簧的刚度特性，调节$c$可以改变阻尼器的阻尼特性。而质量$M$描述了系统的惯性特性。

:::tip
我们需要做一些数学模型抽象的时候，可以把物体间接触分为4个阶段[4]：

1. 物体接近
2. 碰撞振荡
3. 阻尼振荡
4. 稳定

不同阶段可以使用不同的力位控制策略。
:::

### 位置控制(阻抗控制)

反之，阻抗控制就是用力来控制位置。但这种控制方式需要用到电流环，来控制力/力矩。

阻抗模型：

$$
M\ddot{x}+C\dot{x}+Kx=F
$$

我们需要测量的东西是位置和力，需要控制的东西是力/力矩。根据实际位置和期望位置的偏差，得出期望力，通过雅可比矩阵，将期望力转换为关节力矩，通过电流环控制发送出去。

需要购买带电流环的控制器才能实现，所以阻抗控制的成本稍贵。

### 力位混合控制

现在我们搞清楚了*为什么对输出力进行控制，那么你就无法控制位置偏差；反之，你无法控制位置*。

那么，力位混合控制是怎么一回事呢？

我们引入一个坐标系：

当指定力控参考坐标系的一些方向是力控制，另一些方向是位置控制时，就叫做力位混合控制。

这个方法的核心思想是任务空间分解。

因为在不同坐标轴上面做不同的工作，这种控制方法也分为很多分支：

1. 并行的力位混合控制：鲁棒，精度可能差一点
2. 基于任务空间分解的选择性控制：精度高，现在也可以用强化学习来做
3. 自适应力位混合控制：很早就使用神经网络来做了，一样是比较好用的

等等。

我们着重介绍具有代表性的第二种。

先说明一二的区别：并行力位混合控制的两套力/位控制器是独立并行的，基于任务空间分解的选择性控制的两套力/位控制器是基于选择矩阵，选定某个自由度进行力/位控制，任务空间在物理逻辑上划分为**两个互补正交的子空间**。


#### 基于任务空间分解的选择性控制

我们欣赏一下论文[3]的思路：

![](https://github.com/user-attachments/assets/4ef2e763-1bda-4ca7-b1c4-8e5b89e96682)

> 这张图转自\[3\]，侵删，也可以看看原文。

整个控制器的输出力矩 $\tau$ 由两部分**线性叠加**而成：
$$
\tau = \tau_p \text{ (位置/速度控制部分)} + \tau_f \text{ (力控制部分)}
$$

- **位置/速度控制部分** ($\tau_p$)：负责跟踪期望的笛卡尔位置和速度。
- **力控制部分** ($\tau_f$)：负责跟踪期望的笛卡尔力（包含力前馈）。

位置/速度控制部分：

这部分输入是**期望末端位姿** $x_d$ 和**期望末端速度** $\dot{x}_d$。


1. **偏差计算**：
  - 位置偏差：$x_d - T(q)$ （当前末端位姿由正运动学$T(q)$得到）
  - 速度偏差：$\dot{x}_d - J \dot{q}$ （当前末端速度由雅可比$J$和关节速度$\dot{q}$得到）
2. **选择矩阵** $C_p$：这是一个对角矩阵（元素为0或1）。**它决定了在笛卡尔空间的哪个自由度上，位置/速度控制是“激活”的**。被激活的自由度，位置/速度环才会起作用。
3. **雅可比转换**：$(C_p J)^{-1}$ 将**笛卡尔空间**的偏差转换到**关节空间**，以便用关节力矩来修正。
4. **PID调节**：转换后的偏差经过P+PI串级运算，最终得到关节力矩 $\tau_p$。

$$
\tau_p = (K_{pp} + K_{pi}/s) (C_p J)^{-1} (x_d - T(q)) \quad \text{(位置环PI)} + K_{pd} (C_p J)^{-1} ( \dot{x}_d - J \dot{q} ) \quad \text{(速度环P)}
$$

公式中位置环用了PI，速度环用了P。

力控制部分：

这部分输入是**期望力** $F_d$ 和**当前实测力** $F$（由力传感器得到，乘以$K_{fb}$转换）。

1. **力偏差**：$F_d - K_{fb} F$ （期望力减去反馈力）
2. **选择矩阵** $C_f$：与$C_p$类似，但作用于力控制自由度。通常 $C_f = I - C_p$，即力控和位置/速度控的自由度是互补的，不会冲突。
3. **力前馈**：$C_f F_d$ 直接作为前馈项，提高力响应速度。
4. **雅可比转换**：$J^T$ 将笛卡尔空间的力/力矩转换为关节空间的等效关节力矩（虚功原理）。
5. **PI调节**：对力偏差进行比例-积分运算后，再通过$J^T$转换到关节空间。


$$
\tau_f = (K_{fp} + K_{fi}/s) \; J^T C_f (F_d - K_{fb} F) \quad \text{(力偏差PI环)} + J^T C_f F_d \quad \text{(力前馈)}
$$

:::tip
为了保证位置控制和力控制不在同一自由度上相互冲突，两个选择矩阵必须满足**互补正交**关系：
$$
C_p + C_f = I \quad \text{(单位矩阵)}
$$
- $C_p$ 中对角线为1的自由度，由位置/速度环主导。
- $C_f$ 中对角线为1的自由度，由力环主导。
- 一个自由度不能同时被两者控制。
:::


论文[5]也是早期的用选择矩阵来做的力位混合控制。

#### UniFP: 无需力传感器的力位混合控制

这是2025年拿CoRL奖的论文, 用强化学习来做力位混合控制。实际上为啥需要力反馈，又可以把力传感器扔掉呢，因为引入了一个状态估计的环节。

从机器人历史状态出发，通过强化学习估计力，借助位置与速度调整进行补偿，把力、位置、扰动等都模拟出来。从而实现位置跟踪、施力、力跟踪和柔顺交互等多种操作行为。

相关工作可以看一下他们的github链接：https://unified-force.github.io/

PPO实现：

```py
# 1. 计算新旧策略比率的截断损失（Surrogate loss）
ratio = torch.exp(actions_log_prob_batch - old_actions_log_prob_batch)
surrogate = -advantages_batch * ratio
surrogate_clipped = -advantages_batch * torch.clamp(ratio, 1-clip_param, 1+clip_param)
surrogate_loss = torch.max(surrogate, surrogate_clipped).mean()

# 2. 价值函数损失（Value loss，可选 clipped）
value_loss = (returns_batch - value_batch).pow(2).mean()   # 或 clipped 版本

# 3. 熵奖励（Entropy bonus）
loss = surrogate_loss + value_loss_coef * value_loss - entropy_coef * entropy_batch.mean()

# 4. 梯度更新
self.optimizer.zero_grad()
loss.backward()
nn.utils.clip_grad_norm_(self.actor_critic.parameters(), self.max_grad_norm)
self.optimizer.step()
```

上述代码是PPO的更新策略，优化策略网络和值网络。。

看最重要的状态估计模块：

状态估计模块的目的是让策略能够从观测中推断出环境的隐状态（如地形参数、机器人本体状态），从而提升适应性。该模块的训练与PPO策略更新同时进行，代码中体现在：

状态估计模块由编码器和解码器构成：
```py
# 编码器：观测 -> 隐变量 latent
self.adaptation_encoder_module = nn.Sequential(...)   # 输入 self.num_obs，输出 self.num_latent_dim

# 解码器：隐变量 -> 预测状态 obs_pred
self.adaptation_decoder_module = nn.Sequential(...)   # 输入 latent，输出 self.num_obs_pred
```

动作生成天然依赖状态估计产生的隐变量。

动作生成时隐式使用：

调用 self.alg.act(obs, privileged_obs, obs_pred) → 最终进入 ActorCritic.act() → update_distribution() 中：
```python
latent = self.adaptation_encoder_module(observations)          # 编码器
mean = self.actor_body(torch.cat((observations[:, -self.num_obs_now:], latent), dim=-1))
```
即 每个动作都依赖于编码器输出的隐变量。

显式获取隐变量：
在 rollout 循环中，OnPolicyRunner.learn 里：
```python
with torch.no_grad():
    latent = self.alg.actor_critic.get_student_latent(obs)   # 仅用于记录或调试

get_student_latent 会运行编码器+解码器，返回预测的 obs_pred。
```
存储真实标签：

`obs_pred`（真实状态，来自环境特权信息）被存入 `RolloutStorage` 的 `observation_preds` 中，用于后续训练适应模块。

```py
self.transition.observation_preds = obs_pred
```

在 ``act()`` 方法中，将 ``obs_pred``（真实状态标签或目标值）存入 ``transition``.

然后更新阶段训练状态估计模块。

在 update() 方法中，额外的一个训练循环：
```python

if len(self.adaptation_labels) > 0:
    for epoch in range(num_adaptation_module_substeps):
        # 从当前观测推断潜变量（学生网络输出）
        adaptation_pred = self.actor_critic.get_student_latent(obs_batch)
        with torch.no_grad():
            adaptation_target = obs_pred_batch   # 真值标签

        # 对每个标签维度分别计算加权MSE损失
        adaptation_loss = 0
        for label, length, weight in zip(...):
            idx_loss = F.mse_loss(adaptation_pred[:, sel] * weight,
                                  adaptation_target[:, sel] * weight)
            adaptation_loss += idx_loss

        # 只更新适应模块的参数（通常独立于策略网络）
        self.adaptation_module_optimizer.zero_grad()
        adaptation_loss.backward()
        self.adaptation_module_optimizer.step()
```
其中 `self.actor_critic.get_student_latent(obs_batch)`就是从当前观测中估计隐状态的核心调用，而 obs_pred_batch 是训练时提供的真值（例如通过特权信息获取的 terrain stiffness 等）。通过最小化预测与真值的 MSE，状态估计模块学会从观测中提取环境状态。
