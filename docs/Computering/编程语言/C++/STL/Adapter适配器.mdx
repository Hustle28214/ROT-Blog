适配器（Adapter）是一种设计模式，用于将一个类的接口转换成客户希望的另一个接口，使得原本因接口不兼容而不能一起工作的类可以协同工作。在 C++ 标准库中，适配器思想体现在多个方面，主要包括容器适配器、迭代器适配器和函数适配器。

## 1. 容器适配器

容器适配器是对底层序列容器（如 vector、deque、list）的接口进行限制，提供特定的数据结构和操作。它们不提供迭代器，也不支持随机访问，只允许通过受限的接口操作元素。


| 适配器 | 默认底层容器 | 核心操作 | 数据结构特性 |
|--------|--------------|----------|--------------|
| stack | deque | `push()`、`pop()`、`top()` | 后进先出（LIFO） |
| queue | deque | `push()`、`pop()`、`front()`、`back()` | 先进先出（FIFO） |
| priority_queue | vector | `push()`、`pop()`、`top()` | 最大堆（默认），元素按优先级排序 |

底层容器要求：

1. `stack`：支持 `back()`、`push_back()`、`pop_back()` 的容器（如 deque、vector、list）。

2. `queue`：支持 `front()`、`back()`、`push_back()`、`pop_front()` 的容器（如 deque、list）。

3. `priority_queue`：支持随机访问迭代器、`front()`、`push_back()`、`pop_back()` 的容器（如 vector、deque），且元素必须可比较。

容器适配器只是封装，不提供迭代器，因此不能使用范围 for 循环或算法直接遍历。

```cpp
#include <iostream>
#include <stack>
#include <queue>
#include <vector>

int main() {
    // stack 使用默认 deque
    std::stack<int> st;
    st.push(1);
    st.push(2);
    std::cout << "stack top: " << st.top() << '\n';  // 2

    // queue 使用 list 作为底层容器
    std::queue<int, std::list<int>> q;
    q.push(10);
    q.push(20);
    std::cout << "queue front: " << q.front() << '\n'; // 10

    // priority_queue 使用 vector 作为底层容器，自定义比较（最小堆）
    std::priority_queue<int, std::vector<int>, std::greater<int>> pq;
    pq.push(5);
    pq.push(1);
    pq.push(10);
    std::cout << "priority_queue top: " << pq.top() << '\n'; // 1

    return 0;
}
```

## 2. 迭代器适配器（Iterator Adapters）

迭代器适配器是预定义的迭代器，它们包装另一个迭代器或容器，改变迭代行为或提供新的功能。它们通常通过工厂函数创建，并可与算法无缝协作。
