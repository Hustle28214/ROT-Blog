`std::deque`也就是双端队列，可以让用户从队列的头部和尾部进行插入和删除操作，同时可以随机访问（`push_front`、`pop_front`、`push_back`、`pop_back`）。

支持通过下标 `[]` 或 `at()` 直接访问任意位置的元素，时间复杂度为`O(1)`。

```cpp
#include <iostream>
#include <deque>

int main() {
    std::deque<int> dq;

    // 尾部插入
    dq.push_back(1);
    dq.push_back(2);
    // 头部插入
    dq.push_front(0);
    dq.push_front(-1);

    // 随机访问
    std::cout << dq[2] << std::endl;   // 输出 1

    // 遍历
    for (auto x : dq) {
        std::cout << x << " ";          // -1 0 1 2
    }
    std::cout << std::endl;

    // 删除头部和尾部
    dq.pop_front();
    dq.pop_back();                       // 现在为 0 1

    return 0;
}
```

## 实现原理

deque 的典型实现是**分段连续线性空间**。它由一个中控器（map） 和多个缓冲区（buffer） 组成：

- 缓冲区（buffer）：是固定大小的连续内存块，通常大小由实现定义（例如 512 字节）。每个缓冲区存储一定数量的元素。

- 中控器（map）：是一个指针数组（或类似结构），每个元素指向一个缓冲区。当需要扩容时，只需向 map 中添加新的缓冲区指针，而不必移动已有元素。

这种设计使得 deque 在逻辑上看起来是连续的，但物理上是分段的。因此，在两端插入删除时，只需要在头部或尾部缓冲区操作，或者当该缓冲区满时，分配新的缓冲区并更新中控器，无需移动其他元素。随机访问时，通过中控器和偏移量计算元素位置：首先确定元素位于哪个缓冲区，然后在该缓冲区内的偏移量。

优点是，避免了 vector 在头部插入/删除时的大量元素移动，也避免了 list 无法随机访问的缺点。

缺点是，随机访问比 vector 稍慢（因为多了一次间接寻址），且内存占用略高（需要中控器开销）。

## 常用方法

| 操作 | 时间复杂度 | 说明 |
|------|------------|------|
| push_back | 均摊 O(1) | 在尾部添加元素，若尾部缓冲区有空间则直接插入，否则分配新缓冲区 |
| push_front | 均摊 O(1) | 在头部添加元素，若头部缓冲区有空位则插入，否则分配新缓冲区 |
| pop_back | O(1) | 删除尾部元素，若尾部缓冲区变空可能释放 |
| pop_front | O(1) | 删除头部元素，若头部缓冲区变空可能释放 |
| insert / erase（中间） | O(n) | 在中间插入或删除元素需要移动该位置之后的元素，与 vector 类似 |
| operator[] / at() | O(1) | 随机访问，通过计算缓冲区位置和偏移量 |
| size() / empty() | O(1) | 通常维护计数器 |
| clear() | O(n) | 销毁所有元素，可能释放缓冲区 |


## 迭代器失效情况

deque 的迭代器失效规则比 vector 复杂，但比 list 更严格。在首尾两端进行插入或删除时，只有指向被操作元素的迭代器失效，其他迭代器（包括指向另一端元素的）仍然有效。但在某些实现中，如果插入导致缓冲区重新分配，则所有迭代器都可能失效（标准未强制规定，但常见实现是只影响首尾端，不影响另一端）。在中间插入或删除元素时，所有迭代器都会失效（因为元素移动会导致迭代器指向的地址变化），类似于 vector。

因此，在使用 deque 时，如果需要对两端操作并保持迭代器有效，应避免在中间插入删除。

## 功能对比

| 容器 | 随机访问 | 头部插入/删除 | 尾部插入/删除 | 中间插入/删除 | 内存布局 | 迭代器失效规则 |
|------|----------|---------------|---------------|----------------|----------|----------------|
| vector | O(1) | O(n) | 均摊 O(1) | O(n) | 单块连续内存 | 重分配时全部失效；插入/删除点之后迭代器失效 |
| deque | O(1) | 均摊 O(1) | 均摊 O(1) | O(n) | 多块连续内存（分段连续） | 插入/删除点之前的迭代器可能有效，但通常两端操作不影响已有迭代器（除首尾块变化外） |
| list | 不支持（需遍历） | O(1) | O(1) | O(1)（已知位置） | 双向链表，非连续 | 除被删除元素外，其他迭代器始终有效 |

## 适用场景与注意事项

实现队列（FIFO）时，需要两端操作（push_back + pop_front）或双端队列（DEQ，如滑动窗口）。

需要随机访问，但也要在头部频繁插入删除（如任务调度器）。

缓冲区管理器（例如生产者-消费者模型中的循环缓冲区）。

注意事项：

1. deque 的 size() 可能需要遍历所有缓冲区来计算，但标准要求常数时间，因此实现通常维护大小计数。

2. 由于分段存储，deque 的迭代器、引用和指针在插入删除时的失效规则较复杂，编程时需留意。

3. 当容器元素很大时，deque 的缓存局部性不如 vector（因为元素分布在不同的内存块中），可能影响遍历性能。
